# Frontend README

このディレクトリは、営業ヒアリングロープレAIサービスのSPA（React + React Router）を実装します。  
開発演習スコープでは、フロントエンドはローカル（localhost）で動作させます。

## フロントエンドの責務

- 画面提供
  - プログラム作成（分野、総ヒアリング回数）
  - ヒアリング事前設定（その回の目標、制限時間）
  - ヒアリング実施（PTT操作、残り時間表示）
  - 評価一覧（タイムスタンプ、セッションタイトル）
  - 評価詳細（複数先輩からの評価表示）
  - 総評ページ（全回終了後に総評と真の課題を表示）
- 音声インタラクション
  - PTTで音声を収録し、バックエンドへストリーミング送信
  - バックエンドから返るAI顧客音声を再生
- 表示制御
  - 「真の課題」は全回終了まで非表示にする
  - 異常終了したセッションは無効扱いとして再実施導線を出す
- API連携
  - バックエンドAPI / WebSocket（ALB配下のエンドポイント）を利用
  - 先輩向け評価入力ページそのものはバックエンド側管理（UUID URL）

## バックエンドに依存する契約（共有事項）

- セッション終了後、バックエンドで以下が確定・更新される前提でUI表示する
  - 文字起こし、通し録音、セッションタイトル
  - CustomerState（気づき度、ラポール度、開示済み情報、各回サマリ）
- 評価一覧/詳細は、HULFT Square書き戻し後にバックエンドDBへ反映されたデータを参照する
- 全回完了時のみ、真の課題と総評を開示する
- 機密情報（APIキー等）はフロントに保持しない

## このREADMEを読むべきタイミング

- フロントエンドの画面実装・状態管理・UXを変更するとき
- APIレスポンスを受けた表示ルール（真の課題の表示条件など）を変更するとき
- PTT/音声再生/WebSocketまわりのクライアント実装を変更するとき

## 開発コマンド

### AWS バックエンド接続（推奨）

フロントは `localhost:5173`、API / WebSocket は AWS ECS ALB へプロキシします。手元で backend を起動する必要はありません。

```bash
cd frontend
npm.cmd install   # または pnpm install
npm.cmd run dev   # または pnpm dev
```

設定は [`/.env.development`](./.env.development) の `VITE_DEV_BACKEND_URL` を参照してください。  
ECS タスクの `CORS_ORIGINS` に `http://localhost:5173` と `http://127.0.0.1:5173` が含まれている必要があります。

ブラウザ: `http://127.0.0.1:5173`

### ローカル backend を使う場合（任意）

```bash
# backend ディレクトリで Postgres + uvicorn を起動後、
# frontend/.env.development.local に以下を設定
# VITE_DEV_BACKEND_URL=http://127.0.0.1:8000
```
